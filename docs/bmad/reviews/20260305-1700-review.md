## Details

### Probleme #1 — Migration v27 manquante (CRITIQUE)
Le schema passe de v26 a v28. Le fichier migrations.ts ne contient qu'une migration toVersion: 28 qui ajoute tutorial_completed a users. Les colonnes superset_id, superset_type, superset_position sur session_exercises n'ont aucune migration. Sur un upgrade, WatermelonDB crashera car les colonnes n'existent pas dans la DB SQLite existante.

Fix: Ajouter une migration toVersion: 27 avec addColumns pour les 3 colonnes superset dans session_exercises.

### Probleme #2 — Couleur hardcodee
Le style quickStartGoText utilise color: '#FFFFFF' au lieu de colors.primaryText. Violation de CLAUDE.md section 3.1.

### Probleme #3 — Email non reset (RGPD)
Le handler handleDeleteAllData reset le nom, XP, level, streaks, etc. mais ne reset pas user.email. L'email est une donnee personnelle — devrait etre reset a null.

### Probleme #4 — Textes hardcodes SS/CIR
Dans SessionExerciseItem.tsx, les labels de groupe superset utilisent des chaines hardcodees 'SS' et 'CIR' au lieu de cles i18n.
