# Rapport — Stats UX — Groupe B — 20260222-1430

## Objectif
Corriger le chevauchement des dates sur l'axe X du BarChart dans l'onglet Volume d'entrainement. Les labels de semaines ("22/2", "15/2", etc.) se superposent car il y a 12 semaines affichées.

## Fichiers concernés
- `mobile/src/screens/StatsVolumeScreen.tsx`
- `mobile/src/model/utils/statsHelpers.ts` (lecture seule pour comprendre le format des weekLabel)

## Contexte technique
- Le BarChart utilise `react-native-chart-kit` (composant `BarChart`)
- Les labels viennent de `stats.perWeek.map(w => w.weekLabel)` — format `"22/2"` (jour/mois)
- 12 semaines = 12 labels sur un axe étroit = chevauchement
- `react-native-chart-kit` BarChart ne supporte PAS la rotation des labels
- Solutions possibles :
  1. N'afficher qu'un label sur 2 ou sur 3 (labels intermédiaires = `""`)
  2. Utiliser un format plus court (ex: juste le jour `"22"` avec le mois en début)
- Le `chartConfig` est créé via `createChartConfig()` depuis `theme/chartConfig.ts`
- `screenWidth - spacing.md * 2` est la largeur du chart

## Etapes
1. Lire `StatsVolumeScreen.tsx` — comprendre comment `chartData.labels` est construit
2. Modifier la construction des labels pour n'afficher qu'un label sur 3 (et `""` pour les autres) :
   ```tsx
   labels: stats.perWeek.map((w, i) => i % 3 === 0 ? w.weekLabel : ''),
   ```
3. Vérifier visuellement que les labels ne se chevauchent plus (12 semaines → 4 labels visibles)
4. Vérifier avec `npx tsc --noEmit`

## Contraintes
- Ne PAS modifier `statsHelpers.ts` — les données sont correctes, c'est l'affichage qui pose problème
- Ne pas changer la logique de calcul des volumes
- Garder les props existantes du BarChart (`fromZero`, `showValuesOnTopOfBars`, etc.)

## Criteres de validation
- `npx tsc --noEmit` -> zero erreur
- Les labels ne se chevauchent plus visuellement
- Les données affichées restent les mêmes (12 semaines)

Aucune dépendance
