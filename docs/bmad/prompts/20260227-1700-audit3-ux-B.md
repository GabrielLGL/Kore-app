## Problèmes identifiés et corrections

### 1. CLS — Mot animé à largeur variable (CRITIQUE)
**Problème :**
Le `<h1>` contient un `<span>` qui alterne entre "CORPS." (6 chars) et "POTENTIEL." (10 chars). Chaque changement de mot redimensionne le span → **layout shift** → mauvais score CWV (Cumulative Layout Shift).

**Actuel :**
```tsx
<span
  className="shimmer-text"
  style={{
    display: "inline-block",
    opacity: phase === "idle" ? 1 : 0,
    transform: ...,
    transition: ...,
  }}
>
  {WORDS[wordIdx]}
</span>
```

**Fix :** Ajouter `minWidth` calculé sur le mot le plus long. Le mot le plus long est "POTENTIEL." (10 chars). À `clamp(2.5rem, 8vw, 5rem)` de font-size, il faut un `minWidth` suffisant. Utiliser `min-width: 11ch` (11 caractères × 1ch) :

```tsx
<span
  className="shimmer-text"
  style={{
    display: "inline-block",
    minWidth: "11ch",
    opacity: phase === "idle" ? 1 : 0,
    transform: ...,
    transition: ...,
  }}
>
  {WORDS[wordIdx]}
</span>
```

### 2. Libellé stat "0€ Pour commencer" — confusion freemium
**Problème :**
La stat "0€ / Pour commencer" est vraie (plan gratuit existe) mais peut induire en erreur : l'utilisateur peut croire l'app entièrement gratuite alors qu'un plan Pro à 2,50€/mois existe.

**Fix :** Changer le libellé en "Freemium" (mot universel du secteur) ou "Plan gratuit inclus" :
```tsx
{ value: "0€", label: "Plan gratuit" },
```
Cela reste positif tout en étant précis.

---

### 3. `<header>` sans `aria-label`
**Problème :**
Le `<header>` du hero est un landmark sémantique HTML mais sans `aria-label`, les screenreaders annoncent juste "bannière" (label générique).

**Fix :**
```tsx
<header
  aria-label="Présentation de Kore"
  className="relative z-[2] min-h-screen flex flex-col items-center justify-center text-center px-6 py-20"
>
```
