# Rapport — Performance Audit — Groupe A (FlatList) — 20260305-2300

## Objectif
Optimiser toutes les FlatList de l'application pour améliorer les performances de scroll, surtout sur les listes longues (exercices, historique, workout).

## Fichiers concernés
- `mobile/src/screens/ExercisesScreen.tsx` (FlatList L264)
- `mobile/src/screens/WorkoutScreen.tsx` (FlatList L415)
- `mobile/src/screens/ChartsScreen.tsx` (FlatList L145)
- `mobile/src/screens/ExerciseCatalogScreen.tsx` (FlatList L475)
- `mobile/src/screens/SettingsScreen.tsx` (FlatList L718, L747 — time pickers)
- `mobile/src/screens/ProgramDetailScreen.tsx` (FlatList L125)
- `mobile/src/screens/SessionDetailScreen.tsx` (DraggableFlatList L195)
- `mobile/src/screens/ProgramsScreen.tsx` (DraggableFlatList L183)

## Contexte technique
- Aucune FlatList n'utilise actuellement `getItemLayout`, `windowSize`, `maxToRenderPerBatch`, `removeClippedSubviews`, ou `initialNumToRender`.
- `ChartsScreen.tsx:106` — `renderSessionItem` est une simple arrow function, pas wrappée dans `useCallback`.
- `SessionDetailScreen.tsx:198` — `renderItem` est inline (arrow function dans JSX).
- `WorkoutScreen.tsx:423` — `renderItem` est inline (arrow function dans JSX).
- `SettingsScreen.tsx:723,752` — `renderItem` inline pour les time pickers.
- Stack : React Native Expo 52, Fabric (New Arch). Dark mode only. Langue FR.
- Theme : `mobile/src/theme/index.ts` — `colors.*`, `spacing.*`.
- `useColors()` hook pour couleurs dynamiques.

## Étapes

1. **Ajouter des props d'optimisation à chaque FlatList :**
   - `initialNumToRender={15}` (ou adapté à la taille d'écran)
   - `maxToRenderPerBatch={10}`
   - `windowSize={5}`
   - `removeClippedSubviews={true}` (Android seulement via `Platform.OS`)
   - `getItemLayout` si les items ont une hauteur fixe (ExercisesScreen, ExerciseCatalogScreen, SettingsScreen time pickers)

2. **Extraire les renderItem inline en useCallback :**
   - `SessionDetailScreen.tsx:198` — extraire le renderItem inline en `const renderItem = useCallback(...)` avec les dépendances correctes
   - `WorkoutScreen.tsx:423` — idem
   - `SettingsScreen.tsx:723,752` — idem pour les 2 time pickers
   - `ChartsScreen.tsx:106` — wrapper `renderSessionItem` dans `useCallback`

3. **Pour ExerciseCatalogScreen** (liste la plus longue potentiellement — catalogue d'exercices) :
   - Vérifier que la pagination (`loadMore`) est efficace
   - Ajouter `onEndReachedThreshold={0.5}` si pas déjà fait

4. **Pour les DraggableFlatList** (ProgramsScreen, SessionDetailScreen) :
   - `ProgramsScreen.tsx:161` — déjà `useCallback`, OK
   - `SessionDetailScreen.tsx:198` — extraire en useCallback

## Contraintes
- Ne pas casser le drag-and-drop des DraggableFlatList
- Ne pas modifier la logique métier, uniquement les props de performance
- Respecter les patterns existants (useColors, useLanguage, etc.)
- Ne pas utiliser `React.memo` sur les composants qui utilisent `withObservables` (HOC gère déjà)
