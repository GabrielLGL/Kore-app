# Rapport — Rappels d'entraînement — Groupe B (UI Settings) — 20260305-2230

## Objectif
Ajouter une section "Rappels d'entraînement" dans l'écran Settings pour configurer les jours et l'heure des rappels.

## Fichiers concernés
- `mobile/src/screens/SettingsScreen.tsx` — nouvelle section UI
- `mobile/src/i18n/fr.ts` — traductions FR
- `mobile/src/i18n/en.ts` — traductions EN

## Contexte technique

### Architecture existante
- SettingsScreen utilise `withObservables` pour observer le User
- Les préférences sont modifiées via `database.write(() => user.update(...))`
- Theme : `useColors()` hook pour les couleurs dynamiques
- i18n : `useLanguage()` → `{ t, language }`
- Haptics : `useHaptics()` pour feedback tactile
- Pattern toggle : le screen a déjà des switches pour `timer_enabled`, `vibration_enabled`, etc.

### Patterns UI obligatoires (CLAUDE.md)
- Pas de `<Modal>` natif — utiliser Portal pattern (`<BottomSheet>`)
- Couleurs depuis `useColors()` — jamais hardcodées
- Haptics sur chaque interaction
- `keyboardType="numeric"` pour les inputs numériques
- Validation avec `validationHelpers.ts`

### Composants réutilisables disponibles
- `<Button>` — variants primary/secondary/ghost/danger
- `<BottomSheet>` — pour le sélecteur d'heure
- `<AlertDialog>` — confirmations
- `useModalState()` — gestion état modal

### 2. Section UI dans SettingsScreen

**Position :** Après la section Timer, avant la section Données.

**Layout :**
```
┌─────────────────────────────────────┐
│ 🔔 Rappels d'entraînement          │
│                                     │
│ Activer les rappels      [toggle]   │
│                                     │
│ (si activé :)                       │
│                                     │
│ Jours d'entraînement               │
│ [Lun] [Mar] [Mer] [Jeu] [Ven]     │
│ [Sam] [Dim]                         │
│                                     │
│ Heure du rappel                     │
│ [18] : [00]          ← pressable   │
│ (ouvre BottomSheet avec pickers)    │
└─────────────────────────────────────┘
```

**Sélecteur de jours :**
- 7 chips en ligne (style similaire à ChipSelector mais multi-select)
- Chaque jour est un bouton toggle (primary quand sélectionné, ghost sinon)
- Taille compacte, affichage horizontal wrap

**Sélecteur d'heure :**
- Affichage "18:00" pressable
- Ouvre un `<BottomSheet>` avec deux ScrollView/FlatList pour heures (0-23) et minutes (0, 5, 10, ... 55)
- Bouton "Confirmer" pour valider

### 3. Logique de sauvegarde

À chaque changement (toggle, jour, heure) :
```typescript
await database.write(async () => {
  await user.update(u => {
    u.remindersEnabled = value
    u.reminderDays = JSON.stringify(selectedDays)
    u.reminderHour = hour
    u.reminderMinute = minute
  })
})
```

Puis appeler `updateReminders()` du service (Groupe A) pour replanifier.

### 4. Gestion des permissions
- Quand l'utilisateur active le toggle :
  1. `requestNotificationPermission()`
  2. Si refusé → afficher un message inline (pas une alerte), désactiver le toggle
  3. Si accepté → activer et planifier

## Contraintes
- Ne pas casser les sections existantes du SettingsScreen
- Les jours sont stockés en JSON `"[1,3,5]"` (ISO weekday)
- Le sélecteur d'heure doit être accessible (pas de picker natif — incompatible Fabric)
- Animer la section jours/heure avec `LayoutAnimation` quand le toggle s'active
- La section n'apparaît que si `reminders_enabled` est dans le modèle (dépend du Groupe A pour la DB)

## Critères de validation
- `npx tsc --noEmit` → zéro erreur
- `npm test` → zéro fail (ou tests ajustés)
- UI respecte le design system (couleurs theme, spacing, borderRadius)
- Toggle active/désactive visuellement la section
- Les traductions FR et EN sont complètes
- Haptics sur chaque interaction (toggle, sélection jour, changement heure)

## Dépendances
**Dépend du Groupe A** — les champs DB (`reminders_enabled`, `reminder_days`, `reminder_hour`, `reminder_minute`) et les fonctions service (`updateReminders`) doivent exister.
