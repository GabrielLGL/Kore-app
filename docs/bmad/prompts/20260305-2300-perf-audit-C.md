# Rapport — Performance Audit — Groupe C (Mémoire / Memoization) — 20260305-2300

## Objectif
Réduire la consommation mémoire et les re-renders inutiles en ajoutant useMemo/useCallback là où c'est nécessaire, et en vérifiant les cleanups de timers/subscriptions.

## Fichiers concernés
- `mobile/src/screens/ChartsScreen.tsx` — renderItem non memoized (L106)
- `mobile/src/screens/ExercisesScreen.tsx` — vérifier `renderExerciseItem` memoization
- `mobile/src/components/WorkoutExerciseCard.tsx` — debounce timers (L61-62, L84, L90)
- `mobile/src/components/RestTimer.tsx` — setInterval + multiple setTimeout (L107, L127-128, L146)
- `mobile/src/hooks/useWorkoutTimer.ts` — setInterval (L37)
- `mobile/src/screens/ProgramsScreen.tsx` — setTimeout (L115, L286)
- `mobile/src/screens/ProgramDetailScreen.tsx` — setTimeout via ref (L174)
- `mobile/src/screens/ExerciseCatalogScreen.tsx` — setTimeout (L342)
- `mobile/src/navigation/index.tsx` — setTimeout (L95)
- `mobile/src/components/WorkoutSummarySheet.tsx` — debounce setTimeout (L95)

## Contexte technique
- Les timers utilisent des refs (`useRef<ReturnType<typeof setTimeout>>`) — bon pattern.
- Il faut vérifier que CHAQUE timer a un cleanup dans le `useEffect` return ou dans la logique de démontage.
- `withObservables` gère les subscriptions DB automatiquement — pas de leak là.
- Stack : React Native Expo 52, Fabric.

## Étapes

### 1. Audit des timers — vérifier les cleanups

Pour chaque fichier listé, vérifier :
- Le setTimeout/setInterval a-t-il un `clearTimeout`/`clearInterval` dans un useEffect cleanup ?
- Si le timer est dans un event handler (pas un useEffect), le ref est-il clear dans le useEffect unmount ?

Fichiers prioritaires :
- `RestTimer.tsx` — 4 timers (interval + 3 timeouts). Vérifier que le useEffect cleanup les clear TOUS.
- `useWorkoutTimer.ts` — setInterval. Vérifier cleanup.
- `WorkoutExerciseCard.tsx` — 2 debounce timers. Vérifier que le useEffect unmount les clear.
- `WorkoutSummarySheet.tsx` — debounce. Vérifier cleanup.

### 2. Ajouter useCallback/useMemo manquants

- `ChartsScreen.tsx:106` — `renderSessionItem` : wrapper dans `useCallback` avec dépendances `[colors, t]`
- `ExercisesScreen.tsx` — vérifier que `renderExerciseItem` est dans un `useCallback`
- Vérifier que les fonctions passées en props aux composants enfants sont memoizées (évite re-render enfant)

### 3. Vérifier les objets/arrays créés inline dans le JSX

Chercher les patterns comme :
- `style={[styles.x, { backgroundColor: colors.y }]}` — OK (React Native optimise)
- `data={someArray.filter(...)}` ou `data={[...array]}` dans le JSX — PROBLÈME (nouvelle ref chaque render)
- `onPress={() => someFunction(id)}` dans une FlatList renderItem — PROBLÈME (nouvelle ref)

### 4. Vérifier les Animated values

- `AnimatedSplash.tsx` — vérifier que les Animated.Value sont dans useRef, pas dans le body
- `ExercisesScreen.tsx` — `useKeyboardAnimation` (hook custom, probablement OK)

## Contraintes
- Ne pas modifier la logique métier
- Ne pas ajouter de dépendances npm
- Respecter les patterns existants (useColors, useLanguage, useHaptics)
- Ne pas sur-optimiser — `useCallback` uniquement là où ça a un impact mesurable (props passées à des composants memoizés ou renderItem de FlatList)

- `npm test` → zéro fail
- Aucune régression fonctionnelle
- Tous les timers ont un cleanup vérifié
