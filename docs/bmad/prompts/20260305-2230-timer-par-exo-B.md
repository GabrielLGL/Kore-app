# Rapport — Timer par exercice — Groupe B (WorkoutScreen) — 20260305-2230

## Objectif
Modifier le WorkoutScreen pour utiliser le `restTime` de chaque exercice au lieu du timer global. Fallback : `sessionExercise.restTime ?? user.restDuration ?? 90`.

## Fichiers concernes
- `mobile/src/screens/WorkoutScreen.tsx`

## Contexte technique
- **RestTimer** est affiche dans WorkoutScreen (ligne ~461-471)
- **Actuellement** : `duration={user?.restDuration ?? 90}` (ligne 464) — toujours global
- **Trigger** : `handleValidateSet` (lignes 373-400) active `setShowRestTimer(true)` apres validation d'une serie
- **Superset logic** : le timer ne s'affiche qu'apres que TOUS les exercices du superset ont complete le meme set
- **State** : `showRestTimer` (boolean) controle l'affichage
- **Donnees disponibles** : `sessionExercises` est un array observable de `SessionExercise` — chaque element a maintenant `restTime` (apres Groupe A)
- Le composant `<RestTimer>` accepte deja un prop `duration: number`

## Etapes

1. **Ajouter un state pour tracker l'exercice courant** :
   - Ajouter `const [currentRestDuration, setCurrentRestDuration] = useState<number>(user?.restDuration ?? 90)`

2. **Modifier `handleValidateSet`** :
   - Quand le timer doit s'afficher (`setShowRestTimer(true)`), calculer d'abord la duree :
   - Pour un exercice normal : `setCurrentRestDuration(sessionExercise.restTime ?? user?.restDuration ?? 90)`
   - Pour un superset : utiliser le `restTime` du DERNIER exercice valide dans le groupe (celui qui complete le round), ou fallback global

3. **Modifier le `<RestTimer>` render** :
   - Remplacer `duration={user?.restDuration ?? 90}` par `duration={currentRestDuration}`

## Contraintes
- Ne PAS modifier le composant `<RestTimer>` lui-meme — il est deja generique
- Ne PAS changer la logique superset (le timer s'affiche uniquement apres un round complet)
- Garder le fallback : `restTime ?? user.restDuration ?? 90`
- Ne pas toucher aux autres screens

## Criteres de validation
- `npx tsc --noEmit` → zero erreur
- `npm test` → zero fail
- Si `sessionExercise.restTime` est null → comportement identique a avant (timer global)
- Si `sessionExercise.restTime = 120` → le timer affiche 120s au lieu du global

## Dependances
Depend du Groupe A (champ `restTime` dans le model SessionExercise).
