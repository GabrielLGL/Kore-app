# Rapport — Rappels d'entraînement — Groupe A (Backend) — 20260305-2230

## Objectif
Ajouter l'infrastructure backend pour les rappels d'entraînement : schéma DB, modèle User, et service de notifications planifiées.

## Fichiers concernés
- `mobile/src/model/schema.ts` — migration v29
- `mobile/src/model/models/User.ts` — nouveaux champs
- `mobile/src/services/notificationService.ts` — nouvelles fonctions rappels
- `mobile/src/services/__tests__/notificationService.test.ts` — tests

## Contexte technique

### Infrastructure existante
- `expo-notifications@~0.29.14` déjà installé et configuré
- `notificationService.ts` existe avec : `setupNotificationChannel()`, `requestNotificationPermission()`, `scheduleRestEndNotification()`, `cancelNotification()`
- Lazy-loading via `getNotifications()` pour éviter les crashes
- Channel Android `rest-timer` déjà créé

### Schéma actuel (v28)
Table `users` a déjà : `timer_enabled`, `vibration_enabled`, `timer_sound_enabled`, `rest_duration`, etc.
Pas de champs pour les rappels.

### Patterns obligatoires (CLAUDE.md)
- Mutations DB dans `database.write()`
- Schema ↔ Model sync : chaque `@field` doit avoir un column dans schema.ts
- Pas de `any` TypeScript
- Pas de `console.log` sans `__DEV__`
- i18n : utiliser `t()` de `useLanguage()` pour les strings visibles (mais dans le service, les strings de notification sont hardcodées en FR — pattern existant acceptable)

## Étapes

### 1. Migration schema v28 → v29
Dans `schema.ts`, ajouter à la table `users` :
```typescript
// Rappels d'entraînement
{ name: 'reminders_enabled', type: 'boolean' },
{ name: 'reminder_days', type: 'string', isOptional: true },  // JSON array: [1,3,5] (lun, mer, ven)
{ name: 'reminder_hour', type: 'number' },    // Heure (0-23), défaut 18
{ name: 'reminder_minute', type: 'number' },  // Minute (0-59), défaut 0
```

Migration v29 :
```typescript
addColumns({
  table: 'users',
  columns: [
    { name: 'reminders_enabled', type: 'boolean' },
    { name: 'reminder_days', type: 'string', isOptional: true },
    { name: 'reminder_hour', type: 'number' },
    { name: 'reminder_minute', type: 'number' },
  ],
})
```

## Contraintes
- Ne pas casser le rest timer existant (channel séparé)
- Garder le pattern lazy-loading `getNotifications()`
- Les jours utilisent la convention ISO : 1=lundi, 7=dimanche
- expo-notifications utilise weekday 1=dimanche, 2=lundi... → mapper correctement !
- Valeurs par défaut : `reminders_enabled=false`, `reminder_hour=18`, `reminder_minute=0`, `reminder_days='[1,3,5]'`
