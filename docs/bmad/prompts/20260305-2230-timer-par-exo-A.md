# Rapport — Timer par exercice — Groupe A (Fondation DB) — 20260305-2230

## Objectif
Ajouter un champ optionnel `rest_time` (en secondes) à la table `session_exercises` pour permettre un temps de repos personnalisé par exercice. Ce groupe pose la fondation DB sans toucher à l'UI.

## Fichiers concernés
- `mobile/src/model/schema.ts`
- `mobile/src/model/migrations.ts`
- `mobile/src/model/models/SessionExercise.ts`
- `mobile/src/model/models/Program.ts`

## Contexte technique
- **Schema actuel** : v28 (`mobile/src/model/schema.ts`, ligne 4)
- **Table `session_exercises`** : colonnes existantes = `session_id`, `exercise_id`, `position`, `sets_target`, `sets_target_max`, `reps_target`, `weight_target`, `superset_id`, `superset_type`, `superset_position`, `created_at`, `updated_at`
- **Migrations** : fichier `mobile/src/model/migrations.ts`, utilise `schemaMigrations` + `addColumns`
- **Model SessionExercise** : `mobile/src/model/models/SessionExercise.ts`, utilise decorators `@field`, `@relation`, `@date`
- **Program.duplicate()** : `mobile/src/model/models/Program.ts` lignes 20-75 — copie tous les champs d'un SessionExercise. DOIT aussi copier `restTime`.
- **Contrainte CLAUDE.md** : Schema <-> Model sync obligatoire. Chaque `@field` doit avoir un `column` correspondant.

## Etapes

1. **Schema v29** — Dans `schema.ts` :
   - Changer `version: 28` → `version: 29`
   - Ajouter dans la table `session_exercises` : `{ name: 'rest_time', type: 'number', isOptional: true }`

2. **Migration v29** — Dans `migrations.ts` :
   - Ajouter un bloc migration `toVersion: 29` avec `addColumns` sur `session_exercises` : `{ name: 'rest_time', type: 'number', isOptional: true }`

3. **Model SessionExercise** — Dans `SessionExercise.ts` :
   - Ajouter `@field('rest_time') restTime?: number | null` (après `supersetPosition`)

4. **Program.duplicate()** — Dans `Program.ts` :
   - Ligne ~65 (dans le `create` callback du nouveau SessionExercise) : ajouter `newSE.restTime = se.restTime ?? null`
   - Le placer après la ligne `newSE.supersetPosition = se.supersetPosition ?? null`

## Contraintes
- Ne pas casser les migrations existantes (v27, v28)
- Le champ DOIT etre `isOptional: true` — `null` = utiliser le timer global
- Validation : 10-600 secondes (sera faite cote UI, pas ici)
- Ne pas toucher aux screens ou composants UI
